# Res-Contact — Time Breakdown Roadmap (UPDATED for 8-GB MacBook Air M3)

## Objective (by Monday, November 3, 2025)
Deliver a working end-to-end pipeline (CLI + FastAPI) plus a small monitoring notebook that:
1) ingests PDB/mmCIF,
2) builds residue features from ESM2 (MSA optional & budgeted),
3) trains a **low-rank bilinear** contact scorer with **O(P) sampled pairs**,
4) reports PR-AUC / ROC-AUC / Precision@k (with intra vs inter-chain splits).

---

## Constraints & Strategy (8-GB MacBook Air M3)
- **Device:** Prefer Apple **MPS**, fallback to CPU.
- **Model:** Default **ESM2 t6_8M**; cache per-sequence embeddings to disk (`.npz`).
- **MSA:** **OFF by default** (slow servers / rate limits). Optional budgeted online Jackhmmer/BLAST:
  - `MSA_TOTAL_QUERY_BUDGET` (e.g., 10), `MSA_TIMEOUT_S` (e.g., 40s), `MSA_MAX_SEQS` (e.g., 64).
  - If MSA is skipped or budget/timeouts hit → continue with ESM2-only.
- **Memory:** Never build full **L×L** maps in training. Use **sampled pairs** (all positives + random negatives).
- **Complexity:** Per step **O(P·r)** for rank-r bilinear with sampled pairs (P ≪ L²).
- **Streaming:** Iterate chain-by-chain; free tensors; `torch.mps.synchronize()` + `empty_cache()` where helpful.
- **Long proteins:** cap `MAX_LEN_PER_CHAIN` (e.g., 600).

## Method (ESM2 baseline → optional MSA; O(P) optimization)
- **Backbone embeddings:** Per-residue hidden states from **ESM2** (default: `facebook/esm2_t6_8M_UR50D`).
- **Optional MSA → PSSM (budgeted):** Remote Jackhmmer/BLAST with strict query/time budgets.
  - Build frequency matrix **[L×20]** (+entropy) aligned to the query; cache to disk.
- **Contact head (default): Low-Rank Bilinear**
  - Learn projections **U,V ∈ ℝ^{d×r}** (small rank r, e.g., 64–128).
  - Residue i score vs j: **sᵢⱼ = (HᵢU) · (HⱼV) + b** → implicit **rank-r** factorization.
  - **Training:** Score only a **subset of pairs** P per step (all positives + k× negatives).
  - **Complexity:** Project once per chain (O(L·d·r)), then **O(P·r)** for pair scoring.
- **(Optional) Pair-MLP ablation:** On concatenated pairwise features `[hi, hj, |hi−hj|, hi⊙hj, pssm_i, pssm_j]`.

---

## Data Pre-processing
1. **Parse structure:** PDB/mmCIF → chains, sequences (PPBuilder), Cα coordinates.
2. **Seq↔coord alignment:** Stable per-chain residue indexing under `MAX_LEN_PER_CHAIN`.
3. **Contact labels:** Boolean contact by **Cα–Cα < 8.0 Å**; intra by default; inter-chain toggle.
4. **Embeddings:** Tokenize; run ESM2; strip specials; cache `[L, d]`.
5. **(Optional) MSA→PSSM:** Jackhmmer/BLAST (budgeted). Build `[L, 20]` (+entropy).
6. **Pair sampling:** For training, collect **all positives** and **NEG_RATIO** negatives per positive.

---

## Training (O(P) with Low-Rank Bilinear)
- **Sampling per chain:** 
  - Positives: all (or capped) close pairs.
  - Negatives: random pairs at ratio (e.g., 3×); avoid duplicates.
- **Per-step operations:**
  - Compute **A=HU, B=HV** once (O(L·d·r)).
  - For sampled set **P**, score pairs via **r-dim inner products** (O(P·r)).
- **Inter-chain (optional):** Enable **A–B** scoring with rectangular sampling (cap pairs).
- **Loss:** `BCEWithLogitsLoss()` on sampled pairs.
- **Optimizer:** AdamW(lr=1e-3); rank r=128 (tunable).
- **Stability:** Gradient clipping optional; set `seed=42`.

## Evaluation & Post-processing
- **Approx (fast):** sample pairs; compute **PR-AUC / ROC-AUC / Precision@k**.
- **Metrics:** Global + per-PDB; **intra vs inter** reported separately.
- **Artifacts:** `artifacts/metrics.jsonl` (one line/epoch + test summary), CSVs under `results/`.

## Hyperparameters & Training Setup
- **ESM2 variant:** `t6_8M` (default) / `t12_35M` (if resources allow).
- **Bilinear rank:** `r = 128` (try 64/96 for speed).
- **NEG_RATIO:** 3 (try 1–5).
- **TRAIN_BATCH_PAIRS:** 20k (tune 10k–50k).
- **EPOCHS:** 3 (prototype) → 10+ for full runs.
- **Cutoff:** 8.0 Å (Cα).
- **MSA budgets:** `MSA_TOTAL_QUERY_BUDGET=10`, `MSA_TIMEOUT_S=40`, `MSA_MAX_SEQS=64`.
